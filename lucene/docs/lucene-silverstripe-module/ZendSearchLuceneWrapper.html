Retrieves a Zend_Search_Lucene_Interface object connected to the search  index.


If the index does not exist, it is created.  On creation, any callbacks  added using ZendSearchLuceneWrapper::addCreateIndexCallback() are run.  To use this feature, add your callback registration to your _config.php:
  1.  function create_index_callback() {
    2. 

  2.      $index = ZendSeachLuceneWrapper::getIndex();
    3. 

  3.      $index->setMaxBufferedDocs(20);
    4. 

  4.  }
    5. 

  5.  ZendSearchLuceneWrapper::addCreateIndexCallback('create_index_callback');
    6. 

The index lives in the directory specified by the $cacheDirectory static.  If the index already exists, it is opened, unless $forceCache is set  true, in which case the existing index is blanked and a new one created  in its place.  This is useful for re-indexing a site.

Indexes a DataObject.


File objects have text extracted from them using subclasses of the  ZendSearchLuceneTextExtractor class.  Any number of text extractors can  attempt to scan a file of any given extension; the first successful one  will be used.
If the dataobject has a function called Link(), this will be added as an  unindexed field.

 Static Method rebuildIndex (line 377)
 
 

Integer rebuildIndex(
)


	

Rebuilds the search index.  Generally called via register_shutdown_function().


When the process starts, a lock file is created in the temp folder to prevent other  reindexes from being started when one is already running.
After each document is indexed, a count is updated in a file in the temp  folder.  This enables the system to keep a count of how many documents  have been indexed so far in a running process, or how many documents  were indexed the last time the reindex process was run.
When the process exits, a second file is created containing the time  taken to reindex.

		
	
Info

	
    
			
  • return - Returns the number of documents that were indexed.
