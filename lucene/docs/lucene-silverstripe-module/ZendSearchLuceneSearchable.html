Located in File: /ZendSearchLuceneSearchable.php

Decorates a DataObject, giving it the ability to be indexed and searched by Zend_Search_Lucene.

To define custom columns, you should this line into your site _config.php:

  1.  ZendSearchLuceneSearchable::enable(array());

This sets up things without defining columns for any classes.

After this, you can define non-default columns to index for each object by using:

  1.  Object::add_extension(
  2.       'SiteTree',
  3.       "ZendSearchLuceneSearchable('Title,MenuTitle,Content,MetaTitle,MetaDescription,MetaKeywords')"
  4.  );

See the docs for the __construct function for advanced config options.

Constructor __construct (line 196)

ZendSearchLuceneSearchable __construct( [ $fieldConfig = null], [ $classConfig = null])

Called by the system when initialising each instance of the decorated class. The argument comes from the Object::add_extension call in the project's _config.php file.

Field configuration can simply be a comma-separated list of fieldnames to scan, or alternatively can be a json-encoded string containing extended config information. Or you can leave it blank.

All keys are optional for each fieldname. The format for $fieldConfig is:

  1.  array(
  2.     'FieldName' => array(
  3.         'name' => 'StoredName',
  4.         'type' => 'keyword|unindexed|unstored|text',
  5.         'content_filter' => [callback]|false
  6.      ),
  7.      [...]
  8.  )

FieldName - the name of the field on the decorated class

name - the name to store this as in the document. Default is the same as the FieldName. The FieldName of 'ID' is a special case, this will always use a name of 'ObjectID' as this is used internally.

type - the type of indexing to use. Default is 'text'.

content_filter - a callback that should be used to transform the field value prior to being indexed. The callback will be called with one argument, the field value as a string, and should return the transformed field value also as a string. Could be useful for eg. turning date strings into unix timestamps prior to indexing. A value of false will indicate that there should be no content filtering, which is the default.

Class-level configuration should be a json-encoded string containing key-value pairs for the class-level config options. The format for $classConfig is:

  1.  array(
  2.      'index_filter' => 'WHERE clause'
  3.  )

index_filter - a string to be used as the second argument to DataObject::get(). An example of how this might be used in a complex situation:

'"ID" IN ( SELECT "ID" FROM "This" LEFT JOIN "Other" ON "This"."ID" = "Other"."ThisID" WHERE "Other"."ThisID" IS NOT NULL )'
